http_method는 HTTP 요청의 메서드를 나타낸다. 예를 들어, GET, POST, PUT, DELETE 등이 될 수 있다. WAF는 이 필드를 사용하여 특정 HTTP 메서드에 대한 보안 규칙을 적용하거나, 허용되지 않는 메서드에 대한 요청을 차단할 수 있다.
http_url은 HTTP 요청의 URL을 나타낸다. URL은 클라이언트가 서버에 요청하는 리소스의 경로를 나타내는 것으로, 예를 들어 "https://www.example.com/products"와 같은 형식이 될 수 있다. WAF는 이 필드를 분석하여 특정 URL 패턴 또는 정규표현식에 일치하는 요청을 필터링하거나 차단할 수 있다.
http_query는 HTTP 요청의 쿼리 파라미터를 나타낸다. 쿼리 파라미터는 URL 뒤에 "?"로 구분되고, 키-값 쌍으로 구성된 데이터이다. 예를 들어, "https://www.example.com/search?q=keyword&page=1"에서 "q=keyword&page=1"이 쿼리 파라미터 이다. WAF는 이 필드를 분석하여 악성 쿼리 스트링이나 SQL 인젝션 등의 공격을 탐지하고 차단할 수 있다.
http_version은 HTTP 요청의 프로토콜 버전을 나타낸다. 일반적으로 "HTTP/1.0" 또는 "HTTP/1.1"과 같은 형식을 가진다. WAF는 이 필드를 사용하여 특정 버전에 대한 취약점을 탐지하고, 보안 문제를 예방할 수 있다.
http_body는 HTTP 요청의 본문 데이터를 나타낸다. 일반적으로 POST 요청과 함께 전송되는 데이터를 포함한다. WAF는 이 필드를 분석하여 악성 스크립트나 파일 업로드 등의 공격을 탐지하고 차단할 수 있다.
